📚 Documentation

🔗 What is MCP (Model Context Protocol)?

The Model Context Protocol (MCP) is an open-source standard that enables AI applications to securely access external data sources and tools. It acts as a bridge between AI models and real-world systems, allowing for:

  • Secure Integration: Connect AI to databases, APIs, and file systems safely
  • Real-time Data Access: Get live information instead of relying on training data
  • Tool Execution: Allow AI to perform actions like calculations, validations, and transactions
  • Standardized Interface: One protocol works across different AI platforms

This AvaTax MCP Server brings Avalara's powerful tax automation directly to your AI workflow, enabling intelligent tax calculations, address validation, and compliance management through natural conversation.

AvaTax API Reference Groups

Explore the comprehensive AvaTax API capabilities organized by functional area:

▶ Core Tax Operations

calculate_tax - Calculate tax for transactions

Real-time tax calculation with line-item detail, exemptions, and jurisdiction breakdown. Use SalesOrder for quotes/estimates (temporary), SalesInvoice for final transactions (permanent but uncommitted).

create_transaction - Create committed transactions

Generate permanent tax records for compliance and reporting. Creates COMMITTED transactions that will be reported for tax filing.

ping_service - Test API connectivity

Verify credentials and service availability

📖 Swagger Reference: AvaTax Transactions API Documentation

▶ Transaction Management & Lifecycle

list_transactions - List and search transactions

Retrieve transactions with filtering by date, status, customer, and amount. Essential for finding existing transactions before modification.

get_transaction - Retrieve specific transaction

Get detailed information about a transaction by company code and transaction code. Use before making changes.

commit_transaction - Commit transaction for reporting

Move transaction from "Saved" to "Committed" status for tax reporting and compliance. Only uncommitted transactions can be committed.

uncommit_transaction - Uncommit transaction

Change committed transaction back to "Saved" status to allow modifications. Cannot uncommit locked transactions.

void_transaction - Void/cancel transaction

Permanently void a transaction for cancelled orders/invoices. Cannot void transactions reported to tax authorities.

adjust_transaction - Adjust/correct committed transaction

Create new version of committed transaction. Original becomes "Adjusted", new version becomes current. Cannot adjust locked transactions.

change_transaction_code - Change transaction code

Rename transaction code for uncommitted transactions. Useful for correcting codes before commitment.

verify_transaction - Verify transaction compliance

Validate transaction accuracy and compliance with tax rules before processing.

get_transaction_audit - Get transaction audit trail

Retrieve audit information including creation details, processing time, and API call information for compliance.

📖 Swagger Reference: AvaTax Transactions API Documentation

Note: Transaction lifecycle follows: Saved → Committed → Locked → (Voided/Adjusted). Only certain operations are allowed at each stage.
